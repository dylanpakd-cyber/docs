# Claims guardrails

Status: Current guardrail for docs, landing copy, and agent-authored content.

## Core rule

Current:

Do not over-document unfinished product behavior as if it already exists. Use labels:

- Current
- In progress
- Planned

In progress:

- Operator-facing docs can describe setup and review paths as they harden in the repo.

Planned:

- Public trial docs can expand after the trial path is ready and approved.

## Preferred wording

Use:

- built for founders
- serious operator software
- private license access
- built for TikTok first
- built for Mac
- guided onboarding
- runs on real iPhones
- many account workflows
- buy and run it yourself
- call based access

## Avoid wording

Avoid:

- phrases that make SignalRoom sound like phone hosting
- phrases that make SignalRoom sound like managed operations
- phrases that make SignalRoom sound like a broad enterprise social suite
- phrases that make SignalRoom sound like an agency
- invented metrics, testimonials, or scarcity
- promises about platform outcomes
- claims about hidden or evasive behavior

## Required safety line

SignalRoom helps operators make attention workflows visible, paced, logged, and reviewable. It does not guarantee platform or account outcomes.

## Repo docs rule

If the repo does not verify a capability, label it In progress or Planned, or omit it.
