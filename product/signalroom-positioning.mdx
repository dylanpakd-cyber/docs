# SignalRoom positioning

Status: Current copy direction with conservative public publishing boundary.

## Product relationship

Current:

- SignalRoom is the public-facing name and landing layer.
- Warmr is the app, repo, and control-plane implementation.
- SignalRoom docs should point back to Warmr as the canonical source for implementation details.

## Public one-liner

Private software for TikTok operators.

## Public body copy

Run your own TikTok phone setup with the control layer founders and growth teams need. Device lanes, posting runs, account state, logs, cooldowns, and operator approvals in one private cockpit.

## CTA

Book a call to trial SignalRoom on your own setup.

## Lower-page line

Private TikTok ops software for teams running their own iPhone farm.

## Target customer

Current positioning:

- YC-style startups.
- Consumer apps.
- Founders and growth teams testing TikTok.
- Teams already doing manual warmup, posting, approvals, and account checks.
- Teams willing to book a call, trial a license, bring their own phones, and prove it on their setup.

## Offer

Current positioning:

- Book a call.
- Check setup and use-case fit.
- Trial license.
- Run on customer-owned phones.

## Capability language

Use these when they are labeled accurately:

- device lanes
- run control
- posting runs
- account state
- logs
- cooldowns
- operator approvals
- repeatable workflows
- guided onboarding
- private license access
- runs on real iPhones
- built for Mac

## Safety line

SignalRoom helps operators make attention workflows visible, paced, logged, and reviewable. It does not guarantee platform or account outcomes.
