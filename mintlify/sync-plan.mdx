# Mintlify sync plan

Status: Current API discovery. No remote writes were made from this file.

The Warmr repo should remain the canonical source for SignalRoom docs. Mintlify should mirror curated pages from this repo-native tree.

## Official API discovery result

Current:

- The documented Admin API endpoints discovered for updates trigger and poll documentation deployments.
- The documented Agent API endpoint, `POST /v2/agent/{projectId}/job`, can create asynchronous agent jobs that edit branches and create pull requests when changes are made.
- The documented Assistant discovery endpoints can search and fetch page content for an already-published docs site, but they require an assistant API key prefixed with `mint_dsc_` and a domain identifier, not the server-side admin key and project id alone.
- No official direct deterministic page or content create/update/upsert endpoint was found in the inspected docs.

Because of that, do not invent page-write behavior against the Admin API.

## Safe publication options

Current safe options:

1. Connect Mintlify to a repo or docs branch that contains curated docs.
2. Let Mintlify rebuild from repo content using its documented update trigger if the project already points to the correct repo source.
3. Use a Mintlify Agent job only for draft branch or pull-request work when the operator explicitly approves that async workflow.

## Exact safe command shape

Only run the update trigger after confirming the Mintlify project already points at the repo/branch containing these docs and after the operator approves remote publication:

```bash
MINTLIFY_PROJECT_ID="6a03d098944b6e7978191748" \
curl -sS -X POST \
  -H "Authorization: Bearer ${MINTLIFY_ADMIN_API_KEY}" \
  -H "Content-Type: application/json" \
  "https://api.mintlify.com/v1/project/update/${MINTLIFY_PROJECT_ID}"
```

Poll update status with the documented status endpoint, `GET /v1/project/update-status/{statusId}`, for the returned status id. Do not include authorization headers or secret values in logs.

## Stop conditions

Stop before remote writes if:

- The project repo or branch source is unknown.
- The API docs do not document the target write behavior.
- The operation would delete or overwrite unrelated content.
- Public navigation changes are broad or ambiguous.
