# Quickstart

Status: Current plus in-progress operator path. Do not treat this as a public trial-ready install guide yet.

This quickstart is for repo operators and agents who need a safe first path through Warmr. It is intentionally conservative and avoids claiming unverified product behavior.

## Relationship

Current:

- SignalRoom is the public-facing name and landing layer.
- Warmr is the app, repo, and control-plane implementation.
- The Warmr repo is the source of truth for code, local docs, runbooks, and agent rules.

## Prerequisites

Current:

- A Mac host that can run Warmr.app.
- Customer-owned physical iPhones connected to that Mac.
- Device trust and Developer Mode completed before automation work.
- One active owner of each iPhone lane at a time.
- Repo work performed in a fresh worktree from `origin/main`, not a dirty canonical checkout.

In progress:

- A clean trial-onboarding checklist for teams outside the repo owner environment.
- Operator docs for approvals, cooldown review, and failure recovery.

## Start with read-only inspection

Current:

1. Open the Warmr worktree.
2. Read `STATUS.md`, then `CLAUDE.md`, `AGENTS.md`, and `CONTEXT.md`.
3. Confirm worktree freshness before doing issue-scoped work.
4. Use read-only control-plane commands first:

```bash
warmrctl status
warmrctl devices list
warmrctl templates list
warmrctl thread list
```

Use `--json` when an agent needs stable output:

```bash
warmrctl --json status
warmrctl --json devices list
```

## Move to controlled actions only after review

Current:

- The control plane has action methods for thread start/stop, evidence export, and app lifecycle.
- Some action methods require automation to be explicitly enabled.
- Agents should prefer visibility and review before action.

Use action commands only when the operator has approved the run and the lane state is understood:

```bash
warmrctl thread start --configuration-id <CONFIGURATION_UUID>
warmrctl thread stop --configuration-id <CONFIGURATION_UUID>
warmrctl logs --configuration-id <CONFIGURATION_UUID> --follow
```
