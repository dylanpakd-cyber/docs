# Cooldowns

Status: In progress.

Cooldowns are part of the safety framing: visibility, pacing, logs, receipts, review, and controlled runs. Current docs should avoid pretending that every cooldown policy is fully implemented.

## Current evidence

Current:

- Templates expose pacing-related fields such as video count ranges, watch-time ranges, and rates.
- Thread summaries expose elapsed time and simple action counters.
- Logs can be subscribed to and reviewed.

## Operator stance

Current:

- Treat cooldowns as reviewable pacing context, not an outcome guarantee.
- Review lane state and recent logs before starting another run.
- Prefer stopping and inspecting over stacking runs blindly.

In progress:

- Clear cooldown surfaces in the operator cockpit.
- Docs that distinguish template pacing from account-level cooldown policy.

Planned:

- A cooldown review checklist for trial operators.
- Receipts that make recent lane activity easy to inspect.
