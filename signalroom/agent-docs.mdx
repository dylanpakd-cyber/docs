## Non-negotiable repo behavior

Current:

- Do not work in a dirty canonical checkout.
- Create a fresh worktree from `origin/main` for new implementation, docs, handoff, or issue-scoped work.
- Run the repo freshness script before issue-scoped work.
- Stop on stale, diverged, or protected-dirty freshness results.
- Do not add paid hosted CI triggers without explicit approval.
- Do not change release, tag, package, or product claims outside the requested scope.

## Safe control-plane behavior

Current:

- Start with read-only commands.
- Use JSON output for agent parsing.
- Treat action commands as operator-approved actions.
- Capture status, logs, and diffs before reporting completion.

## SignalRoom language boundary

Current:

- SignalRoom is the public-facing name and landing layer.
- Warmr is the repo, app, and control-plane implementation.
- Public docs must not imply the product is trial-ready unless the repo and operator confirm it.
